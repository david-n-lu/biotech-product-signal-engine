# Genecopoeia Signal Engine

Prototype analytics platform for tracking Genecopoeia product and reagent usage signals, scientific citations, customer adoption, competitive share-of-voice, and sales opportunities generated by stored scientific activity.

The app starts empty. Recommendations, lead scores, summaries, metrics, and alerts are generated only from product records, evidence records, and sales records stored in the prototype. The included demo dataset is synthetic and must be loaded explicitly from the UI or API.

## Architecture

- Frontend: browser ES modules in `src/main.js`, served from `index.html` with `styles.css`.
- Backend: zero-dependency Node HTTP server in `server.js`.
- Data model: in-memory repository in `src/platform/store.js`, shaped to match `db/schema.sql`.
- Local persistence: the browser saves products, evidence, and sales records to `localStorage` and restores them into the API after a reload or local server restart.
- Analytics: deterministic modules for matching, ingestion, aggregation, lead scoring, summaries, alerts, and reports under `src/platform/`.
- Source connectors: scheduled connector adapters in `src/platform/sourceConnectors.js` plus in-memory scheduling in `src/platform/connectorScheduler.js`.
- Tests: Node test runner under `tests/`.

This keeps the prototype runnable without dependency installation while preserving clear API boundaries for a future React/TypeScript frontend and PostgreSQL-backed Python or Node service.

## Database Schema

See `db/schema.sql`.

Core tables:

- `products`, `product_synonyms`, `product_competitor_equivalents`
- `evidence_records`, `product_evidence_mentions`, `competitor_mentions`
- `institutions`, `evidence_authors`, `evidence_disease_areas`
- `sales_records`
- `alert_rules`, `generated_alerts`

The schema is normalized for auditability and analytics: product mentions, competitor mentions, evidence provenance, authors, disease areas, and sales records remain traceable to source IDs.

`POST /api/connectors/run` and `POST /api/connectors/:id/run` accept optional `productIds`, `connectorIds`, `searchMode: "standard" | "deep"`, and `perProductLimit` fields. Deep mode is intended for selected-product publication searches.

- `GET /api/export/products-evidence.csv`

Analytics endpoints accept filters: `productId`, `sourceType`, `country`, `applicationArea`, `startDate`, and `endDate`.

The CSV export follows the current selected product context. When a product is selected, the export URL includes its `catalogNumber` and `productName`, the downloaded filename is based on those fields, and the CSV includes `catalogNumber` and `productName` columns in addition to evidence provenance.

The Products CSV export downloads the entire Product Registry into one file, including company, product name, catalog number, RRID, product type, application area, synonyms, competitor equivalents, internal owner, and timestamps.

The Products + Evidence CSV export downloads one combined file for the entire Product Registry. Each row includes product fields plus linked evidence, confidence, review status, matched text, competitors, and provenance; products without evidence are still included with blank evidence fields.

## Evidence Ingestion

Supported source types:

- `publication`
- `patent`
- `trial`
- `grant`
- `protocol`
- `conference_abstract`
- `social_mention`
- `sales_record`

Evidence normalization extracts:

- source title, URL or ID, date
- authors, institution, lab, country
- text snippet
- product mention type
- citation context label
- confidence score
- matched product and competitor mentions
- disease areas

External/API-like records are treated as candidate context until reviewed. Candidate confidence is capped by source type so imported search hits do not become curated proof automatically.

## Source Connectors

The Source Connectors panel can run automated searches on demand, and the server schedules enabled connectors using their configured intervals. Connector hits are imported with:

- `reviewStatus: "candidate"`
- low confidence scores
- connector provenance
- source URL or source ID
- product match provenance

Connector searches are company-qualified by default. For example, a registry product named `CD28` is searched externally as `GeneCopoeia CD28`, and APIs that support structured terms receive this as `GeneCopoeia AND CD28`. This keeps automated imports focused on Genecopoeia-related product usage instead of broad biology-topic searches.

The Product Detail view also has an opt-in **Deep publication search** mode for the selected product. It runs only publication connectors (`PubMed`, `Europe PMC`, and `bioRxiv`), expands cleaned product/company variants, requests up to 10 results per query, and pages PubMed beyond the first result window. These hits are still saved only as low-confidence `candidate` evidence until reviewed.

A scientist must use the review action or `PUT /api/evidence/:id/review` to mark a candidate as `curated` before it becomes curated proof. Rejected candidates remain traceable but are visibly marked as rejected.

Included connectors:

- PubMed publications via NCBI E-utilities
- [Europe PMC RESTful Web Service](https://europepmc.org/RestfulWebService) full-text publications for supplier/product mentions that PubMed abstracts can miss
- [bioRxiv API](https://api.biorxiv.org/) preprints, filtered locally for Genecopoeia/product co-mentions because the public endpoint is date-window based
- USPTO / PatentsView patent search, disabled by default until a JSON patent endpoint or USPTO Open Data Portal access is configured
- ClinicalTrials.gov v2 studies
- NIH RePORTER projects
- OpenAlex protocol-oriented works
- Crossref proceedings/conference metadata
- optional custom social-listening endpoint, disabled until configured

The legacy PatentsView endpoint has transitioned away from the old public JSON behavior and may return an HTML transition page instead of API data. The prototype therefore keeps patent search disabled by default rather than importing unreliable patent evidence or showing raw parser errors.

Social media connectors are disabled by default because production social-listening APIs usually require platform credentials, contractual access, or a vendor endpoint.

## Matching And Scoring

Product matching covers:

- exact product names
- catalog numbers
- RRIDs
- company names
- exact and fuzzy synonyms

Citation context labels:

- `core_method`
- `secondary_mention`
- `comparison`
- `negative_mention`
- `unclear`

Lead scoring is explainable and uses publication frequency, recent mentions, grants, inferred institution size, past purchases, protocol usage, conference activity, and clinical trial activity. Each lead includes top contributing features plus evidence and sales provenance.

## Product Catalog Import And Local Search

The Product Registry supports local JSON or CSV import for GeneCopoeia product catalogs. Imported catalog rows create product registry records only; they do not create evidence and do not imply scientific use.

Minimum product fields:

- `productName`
- `productType`
- `applicationArea`

Recommended fields:

- `company`
- `catalogNumber`
- `rrid`
- `synonyms`
- `competitorEquivalents`
- `internalOwner`

After importing a catalog, use the paginated Product Registry search box to search locally by company, product name, catalog number, RRID, product type, application area, or synonym. The registry result pane is scrollable within each page; use Previous and Next to move through larger catalogs. Press Enter or click a product result to select it; the Product Detail and Evidence Explorer views then show only evidence linked to that product. The browser local workspace stores the last selected product and a per-product detail cache keyed by product ID, so selecting a saved product restores its registry fields and linked evidence context. Use **Run evidence search** on the selected product to query enabled source connectors only for that product, or **Deep publication search** to query publication sources more broadly while keeping all imported hits in the review queue.

## Sales And Marketing Use

The platform helps commercial teams connect scientific activity to action:

- Find products by company, product, catalog number, RRID, or synonym.
- See which institutions, labs, and authors are repeatedly associated with a product.
- Detect citation growth, patent/trial mentions, account reactivation, and competitor surges.
- Prioritize accounts with explainable lead scores and provenance-linked reasons.
- Compare Genecopoeia share-of-voice against named competitor equivalents.
- Export evidence and PDF-ready reports for sales planning, product marketing, and field-science review.

## Auditability Rules

- The default app state is empty.
- Demo data is synthetic and loaded only on demand.
- Summaries and recommendations use stored records only.
- Every metric and lead carries provenance IDs.
- Low-confidence candidate records remain distinct from curated evidence.
